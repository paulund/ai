# Language Reference

Vocabulary for architecture reviews and issues. Use these terms exactly — drift is a bug.

## Core Terms

| Term | Definition | Example |
|------|------------|---------|
| **shallow module** | A module whose interface is nearly as complex as its implementation; the abstraction adds indirection without concentrating complexity. | A 50-line wrapper that just forwards calls to a library. |
| **deep module** | A module with a simple interface hiding complex implementation; the interface is much simpler than what it encapsulates. | A cache layer with a 5-method interface hiding eviction, serialization, and distributed coordination. |
| **deletion test** | Thought experiment: if you deleted this module, would complexity concentrate in callers (keep) or move to callers (remove)? | — |
| **seam** | A designed boundary where two modules meet; a place where behavior can be replaced or intercepted without changing callers. | An interface + adapter pair, or a function pointer passed as dependency. |
| **port** | The abstract interface at a seam — what the caller depends on, independent of any implementation. | `PaymentGateway` interface with `charge()`, `refund()`, `verify()`. |
| **adapter** | A concrete implementation of a port that translates to a specific downstream system. | `StripePaymentGateway implements PaymentGateway`. |
| **in-process** | Synchronous call within the same process (function call, method dispatch). | `order.calculateTax()` |
| **local-substitutable** | Local resource that can be swapped for testing (filesystem, clock, environment). | `Clock.now()` implemented as `SystemClock` / `FixedClock`. |
| **remote-but-owned** | Service your team owns and operates (microservice, managed DB, internal queue). | Your team's `billing-service` gRPC endpoint. |
| **true-external** | Third-party service you don't control (GitHub API, Stripe, OpenAI). | `openai.ChatCompletion.create()` |

## Issue Field Vocabulary

| Field | Required Values |
|-------|-----------------|
| **Recommendation** | `Strong` \| `Worth exploring` \| `Speculative` — exactly one |
| **Category label** | `chore` — mandatory on all issues from this skill |
| **State label** | `planned` — mandatory on all issues from this skill |
| **Execution label** | `afk` (default) \| `hitl` (when target shape or AC are genuinely undecided) |

## Prohibited Labels

Do NOT use on issues from this skill:
- `needs-triage`
- `needs-info`
- `needs-more-info`
- Any label not in the above tables

## Formatting Rules

- **Module names in titles**: kebab-case (`user-service`, `cache-layer`)
- **File paths**: relative to repo root, no leading `./` (`src/lib/cache/redis.ts`)
- **One-line outcome in title**: imperative, outcome-focused (`eliminate retry duplication`, `centralize auth decisions`)
- **AI disclaimer**: Must be the first line of every issue body: `> *This was generated by AI during triage.*`
