# Automated Skills

This documents what skills you will use on a scheduled basis.

## Dev Improve Codebase Architecture
Description: Surface architectural friction and propose deepening opportunities — refactors that turn shallow modules into deep ones. The aim is testability, locality of change, and AI-navigability.
Schedule: Weekly
AFK: Yes
Skills: `/dev-improve-codebase-architecture`
Context: Codebase in the current repo, architectural candidates, and reference docs
Outcome: Architectural improvement opportunities documented in CLAUDE.md and proposed as issues in the current repo

## Github Triage
Description: Triage GitHub issues in the current repo using a label-based state machine.
Schedule: Weekly
AFK: Yes
Skills: `/github-triage`
Context: GitHub issues in the current repo

## Learnt
Description: Sweep recent session transcripts and distil lessons into skills and CLAUDE.md so Claude gets smarter over time.
Schedule: Weekly
AFK: Yes
Skills: `/ai-learnt`
Context: Recent session transcripts
Outcome: Updated skills and/or CLAUDE.md entries capturing validated patterns and corrections.

## Marketing SEO Audit
Description: Audit the marketing site for SEO issues, ranking problems, technical SEO health, and on-page optimisation.
Schedule: Monthly
AFK: Yes
Skills: `/marketing-seo-audit`
Context: Marketing site URL(s)
Outcome: Prioritised SEO findings captured as GitHub issues.

## Ops Monitoring
Description: Review monitoring data and logs for any critical issues or anomalies.
Schedule: Nightly
AFK: Yes
Skills: `/ops-monitoring`
Context: Monitoring data and logs
Outcome: Github issues labeled with `bug` and `needs-triage` for any critical issues or anomalies found during monitoring review.

## Simplify Codebase
Description: Identify and remove unnecessary complexity in the codebase, such as redundant abstractions, dead code, and over-engineered solutions.
Schedule: Weekly
AFK: Yes
Skills: `/simplify`
Context: Codebase in the current repo, architectural candidates, and reference docs
